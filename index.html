Installation

01. Download

Grab the latest meg.js from the button above or the GitHub releases page.

02. Load

In Blockbench, navigate to File → Plugins → Load Plugin from File.

03. Setup

Select the downloaded file. The ModelEngine toolbar will appear automatically in your workspace.

Plugin Documentation

MEG Entity Format

A dedicated Blockbench format for ModelEngine entity authoring. Create a new project and select MEG Entity, or convert an existing Generic project via File > Convert Project.

All plugin tools are scoped to this format and will not appear or function in other formats.

MEG Entity Settings

Accessible via Edit > MEG Entity Settings. Stores per-project metadata used for resource pack export.

FieldDescription
NamespaceMinecraft resource namespace (e.g. mypack)
Texture FolderSubfolder for textures (e.g. entity/mymod)

Mythic Mob IDMythicMobs mob identifier

Validation (Error Scanner)

Open via the MEG Validate button in the left sidebar, or press Y.

  • Cube Errors: Checks for elements exceeding the 112px axis limit.
  • Bone Errors: Validates combined bounding boxes of bone groups.
  • Project Warnings: Scans for missing hitboxes, meshes, and incompatible texture animations.

Variant System

Define multiple visual states for a single model. Use the toolbar dropdown to manage visibility states for "Variant Bones".

  • Create, rename, and remove variants dynamically.
  • Set View as Variant: Saves the current bone visibility to the active variant.
  • Default View: Quickly resets to show only non-variant bones.

Quick Tools

Found in Add Element menu for rapid setup of essential components.

  • Add Hitbox: Creates a standard 8x32x8 hidden hitbox bone.
  • Add Shadow: Creates a standard 16x0x16 shadow bone.
